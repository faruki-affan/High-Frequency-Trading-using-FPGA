System Pipeline Architecture

📡 UART Receiver (Data Ingestion)

The system begins with asynchronous market data ingestion through a UART interface. The receiver samples incoming signals at 16× oversampling rate to ensure robustness against noise and timing skew. Using a 50 MHz system clock, each bit is reconstructed deterministically (~434 clock cycles per bit), ensuring reliable conversion from serial to parallel format.

🧩 Packet Parser (Stream Reconstruction)

Incoming byte streams are assembled into structured price packets using a finite state machine. The parser guarantees data integrity by updating registers only when valid signals are asserted, preventing partial or corrupted reads. Once a full packet is received, it triggers the computation pipeline automatically.

📊 Order Book (High-Speed Cache)

The order book acts as a deterministic memory layer storing the latest market state. Designed for single-cycle access (~20 ns), it ensures that all computations operate on a stable snapshot of data. A gated update mechanism freezes inputs during processing, eliminating race conditions and ensuring consistency.

⚙️ Return Computation Engine

Computes real-time returns using fixed-point arithmetic: r = (p_t − p_{t−1}) / p_{t−1}. Since division is expensive in hardware, the design leverages pipelined divider IP cores with Q16.16 precision scaling, ensuring deterministic latency without floating-point overhead.

📈 Covariance Engine (EWMA)

Implements an Exponentially Weighted Moving Average (EWMA) to compute covariance in real-time: σ_new = α(r1 × r2) + (1 − α)σ_old. This avoids storing historical data, enabling low-latency recursive updates while capturing market co-movement efficiently.

🧠 Jacobi Eigenvalue Solver

Decomposes the covariance matrix to extract principal risk components. The solver uses the CORDIC algorithm for trigonometric computations, avoiding multipliers and enabling efficient hardware execution. A finite state machine iteratively performs matrix rotations until convergence.

🎯 Decision Engine

The final stage interprets eigenvalues to generate trading signals. By analyzing dominant risk factors, the system determines BUY, SELL, or HOLD decisions in a fully deterministic hardware pipeline.

Future Improvements & Latency Optimization

⚠️ Current Limitation

The current system uses UART for data ingestion, which introduces significant latency due to low bandwidth and serial communication overhead. Additionally, the DE-10 Lite FPGA platform does not support high-speed Ethernet interfaces, limiting real-time market feed capabilities.

🚀 Ethernet Integration

Migrating to an FPGA platform with native Ethernet (1G/10G) support would drastically reduce I/O latency and enable direct market data feed handling, similar to real HFT systems.

⚡ PCIe / DMA Acceleration

Integrating PCIe-based communication with DMA transfers can eliminate CPU bottlenecks and enable ultra-fast data movement between host and FPGA.

🧠 Advanced Strategy Layer

Enhancing the decision engine with more sophisticated trading strategies based on eigenvalue analysis and multi-asset risk modeling can improve system intelligence.

📊 Multi-Asset Scaling

Extending the architecture to handle multiple stocks simultaneously will require scalable covariance computation and optimized memory utilization.

⚙️ Higher Clock Optimization

Further pipelining and timing optimization can increase achievable clock frequency, reducing computation latency even further.
